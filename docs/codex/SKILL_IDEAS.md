# Skill Ideas

Potential future repo-specific skills:

- `bootstrap-auditor`
  - verifies README, AGENTS, and Docker docs stay aligned
- `task-slicer`
  - turns approved requirements into implementation task breakdowns
- `worktree-starter`
  - creates task-isolated worktree conventions and checklists
- `review-gate`
  - performs focused readiness checks before merge or handoff

Each future skill should:
- describe its trigger clearly
- minimize assumptions about the product stack
- prefer Docker-executed commands where practical
- preserve human approval gates

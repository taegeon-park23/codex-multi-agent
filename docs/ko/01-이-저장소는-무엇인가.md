# 01. 이 저장소는 무엇인가

[인덱스](README.md) | [다음: 02. 시작 전 준비](02-%EC%8B%9C%EC%9E%91-%EC%A0%84-%EC%A4%80%EB%B9%84.md)

이 저장소는 사람이 승인 중심으로 운영하는 `Codex app -> Codex CLI` 작업 템플릿입니다.

쉽게 말하면, "무엇을 만들지 먼저 계획하고, 사람이 승인한 뒤에만 작은 범위로 실행하는 방식"을 안전하게 반복하기 위한 기반 저장소입니다.

## 이 저장소에 들어 있는 것

- Codex app에서 작업을 시작하고 결과를 검토하는 흐름
- Codex CLI로 계획, 실행, 리뷰를 돌리는 구조
- 승인 전에는 읽기 전용으로 움직이는 기본 규칙
- 승인 후에는 격리된 worktree에서 실행하는 방식
- 실행 결과를 `.codex/runs/<run-id>/`에 남기는 아티팩트 구조
- PowerShell 기반 브리지 스크립트

## 아직 들어 있지 않은 것

- 실제 제품 코드
- 특정 제품 스택 선택
- 웹 전용 프레임워크
- 데이터베이스나 캐시 같은 제품용 서비스
- 특정 도메인 요구사항

즉, 이 저장소는 "무엇을 만들지 이미 정해져 있는 제품 저장소"가 아니라 "앞으로 만들 제품을 안전하게 정의하고 추진하는 운영 기반"입니다.

## 어떤 프로젝트에 쓸 수 있나요?

이 템플릿은 특정 종류의 제품에 묶여 있지 않습니다.

예를 들면:

- 웹 앱
- 데스크톱 앱
- 백엔드 서비스
- 채팅 시스템
- 시뮬레이션
- 게임
- 하이브리드 시스템

중요한 점은 "무슨 기술을 쓸지"보다 "어떻게 안전하게 계획하고 승인할지"를 먼저 잡는다는 것입니다.

## 최종 제품 저장소와 무엇이 다른가요?

### 템플릿 저장소

- 운영 원칙이 중심입니다.
- 계획, 승인, 실행, 리뷰 흐름이 중심입니다.
- 제품 요구사항은 아직 비어 있을 수 있습니다.

### 실제 제품 저장소

- 실제 기능 코드가 들어갑니다.
- 제품 요구사항, 설계, 테스트가 구체화됩니다.
- 선택한 기술 스택에 맞는 구현이 들어갑니다.

실제 사용 시에는 보통 이 저장소를 먼저 포크한 뒤, 자신의 프로젝트 저장소로 발전시켜 나갑니다.

## 왜 바로 구현하지 않나요?

이 저장소의 기본 철학은 "먼저 계획, 나중에 실행"입니다.

바로 구현하면 다음 문제가 생기기 쉽습니다.

- 범위가 너무 커짐
- 승인 없이 변경이 커짐
- 제품 스택이 성급하게 고정됨
- 사람이 결과를 검토하기 어려워짐

그래서 이 저장소에서는 먼저 읽기 전용 계획을 만들고, 사람이 결과를 보고 나서 아주 작은 실행 범위만 승인합니다.

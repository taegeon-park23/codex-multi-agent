# 08. 용어 정리

[인덱스](README.md) | [이전: 07. 자주 하는 실수와 해결법](07-%EC%9E%90%EC%A3%BC-%ED%95%98%EB%8A%94-%EC%8B%A4%EC%88%98%EC%99%80-%ED%95%B4%EA%B2%B0%EB%B2%95.md)

이 문서는 처음 보는 사람이 헷갈리기 쉬운 용어를 짧게 정리한 문서입니다.

## supervisor

정의:
- 작업을 직접 다 수행하는 사람이 아니라, 무엇을 할지 정하고 결과를 읽고 승인하는 사람입니다

예시:
- "이 범위까지만 실행하세요"라고 승인하는 사용자

## worktree

정의:
- 같은 Git 저장소에서 분리된 작업 공간을 하나 더 만드는 방식입니다

예시:
- 승인된 작은 실행을 메인 작업 공간과 분리해서 안전하게 진행하는 별도 폴더

## approval gate

정의:
- 다음 단계로 넘어가기 전에 사람이 명시적으로 승인해야 하는 지점입니다

예시:
- 계획 결과를 본 뒤 실행으로 넘어가기 전 승인

## read-only planning

정의:
- 파일을 바꾸지 않고 계획, 구조, 위험, 다음 단계를 정리하는 단계입니다

예시:
- 제품 요구사항 초안과 작업 분해만 만들고 실제 구현은 하지 않는 상태

## execution handoff

정의:
- 계획 결과를 바탕으로, 승인된 범위만 실제 실행 단계로 넘기는 과정입니다

예시:
- 특정 `plan run id`와 승인 메모를 넣고 작은 범위를 실행하는 것

## run artifacts

정의:
- 각 실행 결과를 기록한 파일 묶음입니다

예시:
- `request.md`, `result.json`, `result.md`, `logs/stderr.log`

## bridge script

정의:
- Codex app의 요청을 Codex CLI 실행으로 연결하고, 결과를 파일로 남기는 PowerShell 스크립트입니다

예시:
- `Invoke-SupervisorPlan.ps1`

## Codex app vs Codex CLI

정의:
- Codex app은 사람이 대화하고 승인하는 화면입니다
- Codex CLI는 실제 작업을 실행하는 엔진입니다

예시:
- 앱에서 요청을 넣고, CLI가 뒤에서 계획 또는 실행을 돌린 뒤, 결과를 다시 앱이 요약합니다

## template repo vs actual product repo

정의:
- template repo는 운영 방식과 시작 구조를 제공하는 저장소입니다
- actual product repo는 실제 기능과 제품 코드를 담는 저장소입니다

예시:
- 이 저장소를 포크해 새 프로젝트의 기반으로 삼고, 그다음부터는 자신의 제품 저장소로 발전시키는 방식

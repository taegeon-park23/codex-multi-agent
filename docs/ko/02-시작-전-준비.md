# 02. 시작 전 준비

이 문서는 첫 실행 전에 무엇을 확인해야 하는지 정리한 준비 체크리스트입니다.

## 준비물 체크리스트

- Windows 11
- Git
- Docker Desktop
- Codex app
- Codex app Local mode
- Codex CLI 설치
- Codex CLI 인증 완료

## 먼저 실행할 확인 명령

아래 명령은 이 저장소의 현재 구조를 기준으로 맞춰져 있습니다.

### 1. Git 확인

```powershell
git --version
```

정상이라면 버전 문자열이 나옵니다.

### 2. Docker 확인

```powershell
docker --version
docker compose version
docker compose config
```

정상이라면:

- `docker --version`에서 Docker 버전이 나옵니다.
- `docker compose version`에서 Compose 버전이 나옵니다.
- `docker compose config`가 현재 `docker-compose.yml`을 읽어 서비스 구성을 출력합니다.

### 3. Codex CLI 확인

```powershell
codex --help
codex exec --help
```

정상이라면 도움말이 출력됩니다.

주의:

- 이 두 명령은 "설치되어 있고 호출 가능한지"를 확인하는 용도입니다.
- 실제 인증이 살아 있는지는 나중에 읽기 전용 계획 실행이 성공하는지로 확인하는 것이 가장 현실적입니다.

### 4. 브리지 스크립트 진입점 확인

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File scripts\orchestration\Invoke-SupervisorPlan.ps1 -?
```

이 명령은 계획용 스크립트가 호출 가능한 상태인지 빠르게 확인하는 용도입니다.

## Codex app에서 수동으로 확인할 것

아래 항목은 명령줄이 아니라 앱에서 직접 확인해야 합니다.

- 저장소 루트를 Codex app에서 열었는가
- Local mode로 열었는가
- Codex app를 작업 지시와 승인 화면으로 사용할 준비가 되었는가

이 저장소에서는 Codex app가 사람과 대화하고 승인을 받는 쪽이고, Codex CLI가 실제 작업을 돌리는 쪽입니다.

## 준비 완료 상태는 어떤 모습인가요?

아래 항목이 모두 만족되면 준비 완료로 봐도 됩니다.

- `git --version`이 정상 출력된다
- `docker --version`이 정상 출력된다
- `docker compose version`이 정상 출력된다
- `docker compose config`가 오류 없이 동작한다
- `codex --help`가 정상 출력된다
- `codex exec --help`가 정상 출력된다
- `Invoke-SupervisorPlan.ps1 -?`가 호출된다
- Codex app에서 이 저장소를 Local mode로 열 수 있다
- 첫 읽기 전용 계획 실행이 실제로 완료되면 Codex CLI 인증 상태도 실전 기준으로 확인된 것이다

## 처음 실행 전에 권장하는 순서

1. 저장소 루트에서 PowerShell을 연다.
2. 위 확인 명령을 순서대로 실행한다.
3. Codex app에서 저장소를 Local mode로 연다.
4. 아직 구현 요청은 하지 않는다.
5. 먼저 읽기 전용 계획 프롬프트를 준비한다.

## 자주 놓치는 점

- Codex app가 열린다고 Codex CLI까지 준비된 것은 아닙니다.
- Docker Desktop이 설치되어 있어도 `docker compose config`를 꼭 확인하는 것이 좋습니다.
- 이 저장소는 제품 런타임을 바로 돌리는 저장소가 아니라 운영 템플릿이므로, 첫 실행은 구현이 아니라 계획이어야 합니다.

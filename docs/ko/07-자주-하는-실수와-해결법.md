# 07. 자주 하는 실수와 해결법

[인덱스](README.md) | [이전: 06. 복붙용 프롬프트 모음](06-%EB%B3%B5%EB%B6%99%EC%9A%A9-%ED%94%84%EB%A1%AC%ED%94%84%ED%8A%B8-%EB%AA%A8%EC%9D%8C.md) | [다음: 08. 용어 정리](08-%EC%9A%A9%EC%96%B4-%EC%A0%95%EB%A6%AC.md)

이 문서는 처음 사용하는 사람이 자주 겪는 실수와 해결 방법을 모아 둔 문서입니다.

## 1. 계획 없이 바로 구현하려고 함

문제:
- 처음부터 기능 구현을 바로 요청한다

왜 문제인지:
- 범위가 너무 커지기 쉽고, 승인 없는 변경이 섞이기 쉽습니다

어떻게 고칠지:
- 먼저 읽기 전용 계획 프롬프트를 사용하고, 작은 실행 단위를 제안받은 뒤 그중 하나만 승인하세요

## 2. 승인 단계를 건너뜀

문제:
- 계획 결과를 제대로 읽지 않고 바로 실행을 허용한다

왜 문제인지:
- 이 저장소의 핵심 안전장치가 사라집니다

어떻게 고칠지:
- `result.md`, `result.json`, 필요하면 `approval-request.md`까지 확인한 뒤에만 실행 여부를 결정하세요

## 3. 한 번에 너무 큰 프롬프트를 던짐

문제:
- 제품 정의, 스택 선택, 구현, 테스트, 배포까지 한 번에 다 요청한다

왜 문제인지:
- 계획도 흐려지고 승인 범위도 모호해집니다

어떻게 고칠지:
- 제품 정의, 작업 분해, 작은 실행 승인 순서로 나누세요

## 4. 포크 없이 바로 실사용 시작

문제:
- 템플릿 저장소를 그대로 실사용 저장소처럼 다룬다

왜 문제인지:
- 프로젝트 기록과 템플릿 기반이 섞여 관리가 어려워집니다

어떻게 고칠지:
- 먼저 포크하고, 자신의 프로젝트 저장소에서 시작하세요

## 5. 웹 전용 구조라고 오해함

문제:
- 이 저장소를 웹 앱 전용 템플릿이라고 생각한다

왜 문제인지:
- 실제 프로젝트 정의를 불필요하게 좁혀 버립니다

어떻게 고칠지:
- 이 저장소는 운영 템플릿이라는 점을 기억하세요. 웹, 채팅, 게임, 시뮬레이션, 데스크톱 도구 등에도 그대로 쓸 수 있습니다

## 6. run artifacts를 확인하지 않음

문제:
- 앱의 짧은 요약만 보고 세부 결과를 확인하지 않는다

왜 문제인지:
- 실제 요청, 실제 프롬프트, 승인 필요 여부, 오류 로그를 놓칠 수 있습니다

어떻게 고칠지:
- 최소한 `result.md`와 `result.json`은 항상 확인하고, 이상하면 `logs/stderr.log`까지 보세요

## 7. Codex app이 CLI 내부를 다 본다고 착각함

문제:
- 앱 화면에 보이는 것만으로 내부 작업 전체를 이해했다고 생각한다

왜 문제인지:
- 이 저장소는 아티팩트를 통해 결과를 확인하는 구조입니다

어떻게 고칠지:
- `.codex/runs/<run-id>/` 폴더를 기준으로 확인하세요. 앱은 supervisor 화면이고, run artifacts가 실제 기록입니다

## 기억하면 좋은 한 줄

이 저장소에서는 "계획 먼저, 승인 후 실행, 결과는 아티팩트로 확인"이 기본입니다.
